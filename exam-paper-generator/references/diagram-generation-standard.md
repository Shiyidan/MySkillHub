# 考试图形生成规范

本规范只在新题需要图形时读取。目标是生成可直接进入正式试卷的作答图，不是教学讲解图、答案提示图或装饰性插图。

## 1. 先建立信息披露边界

在写题干和 SVG 前，先输出 `information_disclosure`：

```json
{
  "textual_givens": ["题干明确给出的条件"],
  "visual_givens": ["只能或适合由图形直接呈现的条件"],
  "derived_or_hidden": ["必须由考生推出、不得直接写进图形的关系"],
  "answer_leak_check": "说明图形没有替考生完成哪一步关键推理"
}
```

- SVG 只能呈现 `textual_givens` 和 `visual_givens`。
- `derived_or_hidden` 不得出现在标签、坐标、尺寸、辅助线、颜色、比例或特殊标记中。
- 不要为了“信息完整”重复标注题干已经清楚给出的每个条件。
- 若图形会直接暴露关键中间量、答案范围或解法，应减少标签、改变呈现方式，或取消图形。

## 2. 再做布局计划

生成 SVG 前先输出 `layout_plan`：

```json
{
  "canvas": {"width": 440, "height": 300, "safe_margin": 24},
  "scale_policy": "to_scale | not_to_scale | partially_semantic",
  "objects": [
    {"id": "object-id", "role": "axis | shape | curve | force | marker", "bbox": [70, 30, 400, 240]}
  ],
  "labels": [
    {"text": "P", "anchor": [260, 112], "align": "start", "must_not_overlap": ["line", "point"]}
  ],
  "style": {
    "palette": "black-white-grey",
    "primary_stroke": 1.8,
    "secondary_stroke": 1.2,
    "label_font_px": 17
  }
}
```

布局必须同时满足：

- 所有对象和文字位于安全边距内；标签不得被裁切。
- 标签与点、线、箭头、阴影边界保持约 6–12 px 的视觉间距。
- 任何线条不得穿过文字；公式标签应放在线或曲线旁的空白区。
- 主图占画布主要区域，四周留白均衡，不把图形挤在一角。
- 同类线条、点、箭头和标签使用一致层级，不靠随机颜色区分。

## 3. 使用正式考试的视觉语言

- 默认白底、黑线、深浅灰填充；除非颜色本身是题目条件，否则禁用蓝色等装饰色、渐变、阴影、圆角卡片和图例装饰。
- 填充只用于表达区域、材料、阴影或集合等题意；普通几何边界和坐标矩形默认不填充。
- 线宽建议：主轮廓 1.6–2.0 px，辅助线 1.0–1.3 px，强调线不超过 2.3 px。
- 标签字号建议 15–18 px；点名、变量使用一致的数学字体或斜体，说明文字使用一致的无衬线字体。
- 不在 SVG `<text>` 中写原始 LaTeX 命令；使用普通字符、Unicode 符号或分开的 `<tspan>`。
- 禁用脚本、动画、`foreignObject`、滤镜和外部字体；SVG 必须由基本矢量元素独立渲染。

## 4. 按图形类型执行

### 坐标图

- 坐标轴有清楚的正方向箭头和轴名；只绘制题目需要的象限。
- 刻度、数值和网格只在解题需要时出现，不为美观补齐。
- 若斜率、角度、长度或面积依赖视觉比例，使用一致坐标比例；否则明确 `not_to_scale`。
- 函数曲线或直线必须在定义域内连续、穿过所有给定关键点，不得被填充区域无理由遮挡。
- 点坐标只有在题面直接给出时才能完整标注；由直线方程或其他条件可推出的坐标分量应留给考生推导。

### 平面与立体几何

- 拓扑、相交、包含和相切关系必须准确；视觉比例不得暗示题目未给出的等长、垂直、平行或角度关系。
- 只在题目给出相应条件时添加直角、等长、平行、角弧或对称标记。
- 非按比例示意图必须在图下标明 `[diagram not to scale]`。

### 物理图

- 物体、接触面、运动方向和受力箭头分层绘制；箭头起点、方向和作用对象明确。
- 单位与物理量只出现一次；不得用箭头长度暗示未给出的大小关系。

### 表格、统计图和实验图

- 列名、轴名、单位、图例和数据一一对应；数据精度与题干一致。
- 不使用会改变读数的透视、三维柱形、平滑曲线或装饰性图标。

## 5. SVG 排版与渲染是生成步骤

1. 根据 `information_disclosure` 和 `layout_plan` 写 SVG。
2. 保存 SVG 后，在题目片段提交前渲染为 PNG 预览，建议按最终显示宽度 480–720 px 检查。
3. 逐项检查：裁切、文字碰撞、线穿文字、字号过小、线宽失衡、比例误导、颜色装饰、条件遗漏、推理泄露。
4. 发现任一问题时直接修改 SVG 和布局计划，再生成题目片段；不得把问题留给最终质量门。

渲染记录至少包含：

```json
{
  "svg_path": "assets/example.svg",
  "preview_path": "artifacts/example.preview.png",
  "render_width_px": 600,
  "collision_check": "passed",
  "clipping_check": "passed",
  "information_disclosure_check": "passed",
  "exam_style_check": "passed"
}
```

## 6. 典型失败与改法

坐标矩形题中，同时标出 `P(t, 2t/3)`、底边 `t` 和高 `2t/3`，会把“由直线关系求高”这一步直接告诉考生。合格做法是只标题面给出的点名、直线方程和必要的 `t` 位置；把 `2t/3` 留作推导结果。图形使用黑白灰，坐标轴有箭头，公式放在直线旁的空白区，任何标签不与点或直线相撞。
