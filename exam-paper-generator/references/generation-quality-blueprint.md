# 增强生成蓝图

本文件用于解决仿真题低质量问题：题干近似、只换数字、图形粗糙、难度照抄。生成任何新题前必须读取本文件，并把要求落实到 `source.generation_blueprint` 和逐题事务产物中。

## 1. 母题解剖：先判断什么不能继承

在写新题前，先写 `source_deconstruction`，至少覆盖：

- 核心知识点：真正要保留的 syllabus code 或知识点。
- 推理骨架：母题需要几步、每步的判断或计算是什么。
- 表层元素：题干语境、人名、物体、数值、单位、变量名、句式。
- 图形元素：图形类型、拓扑结构、标签、比例、视觉线索、题文引用方式。
- 选项陷阱：错误选项分别来自什么误区。
- 难度来源：难在知识点、步骤数、抽象程度、图形读取、隐藏条件、选项迷惑性，还是时间压力。

只允许继承“核心知识点”和“考试风格”。表层元素、数值关系、图形拓扑、选项陷阱和难度标签不能默认继承。

## 2. 结构改造：新题必须换骨架

写 `structural_transformation` 时，必须至少改变下列三类中的两类：

- 条件组织：例如把直接给定改成条件比较、约束筛选、反例判断、图像读取或实验情境。
- 推理路径：例如从直接计算改成分类讨论、从单步代入改成反向推理、从数值求解改成性质判断。
- 干扰项逻辑：错误选项来自新的误区，而不是母题错误选项的数值替换。

以下情况必须返工：

- 题干主句基本不变，只替换数字、变量、单位或名词。
- 求解步骤与母题同构，只是数值不同。
- 正确答案位置或错误选项模式沿用母题。
- 图形题只是重画同一图形，改变标签或数值。

## 3. 数值变化不是新颖性

`changed_values` 只能说明数值如何服务于新结构，不能作为新颖性的主要依据。合格写法应说明：

- 数值如何改变步骤数、边界条件或判断方式；
- 数值如何制造新的合理干扰项；
- 数值如何避免答案模式泄露；
- 数值是否保持考试允许范围和计算负荷。

如果数值变化不影响推理结构，只能视为近重复。

## 4. 图形生成规格

写 `diagram_generation_spec` 时先判断是否需要图形：

- 无图题：说明为什么 `images=[]`、`diagram=null` 合理。
- 有图题：必须写清图形语义、必要标签、单位、比例/坐标、关键关系、SVG 元素和题文引用。

图形题必须满足：

- 图形是解题必要信息，不是装饰。
- 图形中的所有标签、单位、点、线、角、方向、坐标或物理量都可由题文引用。
- SVG/矢量资产应可编辑、可渲染，且 `asset_path` 非空。
- 题干、选项、`images[]`、`diagram` 和解析对同一图形关系的描述一致。
- 图形只呈现题面已知和必要视觉已知；必须由考生推出的关系不得出现在坐标、尺寸、辅助线、颜色或特殊标记中。
- 需要图形时必须继续读取 `references/diagram-generation-standard.md`，产出 `information_disclosure`、`layout_plan`、SVG 和渲染记录。

图形设计产物至少包含：

```json
{
  "diagram_type": "coordinate_graph | geometry | physics | table | none",
  "required_for_solution": true,
  "semantic_objects": ["列出点、线、角、物体、坐标轴、曲线或表格字段"],
  "given_relations": ["列出题面直接给出的图形关系"],
  "derived_relations": ["列出解题中需要推出的图形关系"],
  "labels_and_units": ["列出所有标签和单位"],
  "svg_requirements": ["列出必须绘制的 SVG 元素和可读性要求"],
  "misleading_risk_check": "说明比例、角度、坐标或箭头是否可能误导，以及如何避免",
  "answer_leak_check": "说明哪些推导关系不会直接写进图形",
  "render_preflight": "说明目标显示宽度和碰撞、裁切、考试风格检查方式"
}
```

## 5. 干扰项设计

干扰项必须在题目生成阶段设计完成，而不是最后补写。每个错误选项都必须记录：

- 该选项来自哪一种具体误区；
- 学生会怎样从题干条件走到这个错误答案；
- 该错误与哪个条件、公式、图形关系或逻辑规则冲突；
- 如何用本题条件排除该选项。

不合格写法：

- “计算错误”
- “概念错误”
- “不符合题意”
- “其他选项来自符号错误”
- 英文错因或从母题复制的英文解析片段

合格写法示例：

```json
{
  "option": "D",
  "reason": "把直线斜率 2/3 误读成 3/2，导致矩形高度写反，因此面积方程与图中直线关系冲突。"
}
```

## 6. 难度重估

写 `difficulty_reassessment` 时，不参考母题 difficulty 作为默认答案。独立评估：

- 知识点层级：基础概念、综合应用、跨主题迁移。
- 步骤数：有效推理步数，不是解析文字步数。
- 抽象程度：是否需要从语境、图形、符号或逻辑语言中抽取结构。
- 计算负荷：数值大小、代数复杂度、近似/精确要求。
- 图形负荷：是否需要读图、构造辅助线、识别图像性质或多信息整合。
- 干扰项强度：错误选项是否针对真实误区，是否容易诱发。
- 考试匹配度：是否符合目标考试常见难度和节奏。

难度标签建议：

- `easy`：1–2 个核心步骤，条件直接，干扰项明显，少量或无图形负荷。
- `medium`：2–4 个核心步骤，需要转换表示、整合条件或处理较强干扰项。
- `hard`：多步推理、隐含条件、抽象论证、复杂图形读取或强干扰项组合。
- `composite`：至少两个不同知识点都对求解不可缺少，核心负荷来自跨考点串联。不能仅因步骤多、题干长或同时贴了多个标签就判为复合。

`composite` 表示综合结构，不等于“比 hard 更难”。当题目虽有多个步骤但只依赖一个核心知识点时，仍按实际认知负荷归入 `medium` 或 `hard`；只有多个知识点都真正进入解题链时才使用 `composite`。

如果难度标签与 `solution_trace.steps`、选项陷阱或图形负荷不一致，回到蓝图或题目设计步骤返工。

## 7. 新颖性核验矩阵

新题发布前，用下列矩阵比较母题与新题。至少三项必须有实质变化，且“推理路径”必须变化。

| 维度 | 必须比较 | 失败信号 |
| --- | --- | --- |
| 题干句式 | 主句、设问目标、条件呈现 | 主句几乎相同 |
| 语境对象 | 人物、物体、实验/图形背景 | 只换名词 |
| 数值关系 | 等式、比例、边界、参数 | 只换数字 |
| 推理路径 | 步骤、判断、分类、反推 | 步骤同构 |
| 图形拓扑 | 点线关系、坐标关系、物理方向 | 同图换标签 |
| 选项陷阱 | 每个错误选项的误区 | 错误模式相同 |
| 难度来源 | 难在哪里 | 直接复制母题难度 |

任何失败信号出现时，不允许通过“润色文字”修复，必须回到蓝图重新设计。
