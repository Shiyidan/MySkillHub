# QuizTestPRD

## 使用时机

当用户要求整理或重写 QuizTestDemo 的需求文档为 PRD 时使用，包括：

- 会员权益、试卷管理、诊断测试、在线答题、后台管理等功能文档
- 需要非开发可读、可直接转测试用例的版本
- 需要统一五段式说明，突出业务交互、数据流和边界状态

## 输出规则

1. 以“行为逻辑优先”组织，先讲发生了什么、由谁触发、数据来源与状态变化，再讲展示细节。  
2. 所有段落围绕可验证行为：输入、处理、输出、状态、权限、容错。  
3. 明确状态术语：未开始、进行中、已完成、已结束、已暂停、失败、可重试。  
4. 避免过度的视觉样式说明；仅当样式影响流程（如按钮可见性、状态文案、入口位置导致误操作）时简要体现。  
5. 每个大段落给出可验收化描述，方便 QA 转成测试点。  

## 项目约定

- 关键术语统一：  
  - 试卷类型 `paperType`：`realPaper` / `mockPaper` / `aiPaper`  
  - 考试类型 `examType`（按系统现有枚举与白名单定义）
- 会员与权限：明确角色边界（学生、会员、管理员），注明各角色可见项与可操作项差异。  
- 状态来源：优先写“哪些接口/表提供来源数据”，以及“何时刷新、何时持久化”。  
- 日志与可追溯性：涉及关键提交、提交失败、状态变更要标注记录字段或影响对象。  

## 输出结构（固定 5 大部分）

1. 功能边界与目的  
2. 页面描述  
3. 核心流程  
4. 异常容错  
5. 后续与非目标说明
